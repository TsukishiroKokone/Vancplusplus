# Van Lexicon Variables

> 本文档列出 Van Lexicon C++ 版内置变量。变量支持多轮嵌套解析，默认最多 52 轮。

## 嵌套规则

```text
[重复.[随机数字].🌸]
[计算.[长度.abc].*2]
[随机选择.猫猫|兔兔|[name]]
```

## 变量总览

当前版本按“变量族 = 一个可直接使用的变量名 / 一个参数化变量模板前缀”统计，已提供 **520+ 个不同变量族**，并且按要求把下面五大类都扩展到 **100+ 个变量族/别名族**：

| 分类 | 当前规模 | 实现方式 |
|---|---:|---|
| 参数化工具族 | 100+ | `[前缀参数]` 一次 token 扫描，按前缀分派 |
| 用户与事件变量 | 100+ | 上下文字段别名哈希表，一次 token 扫描 |
| 环境变量 | 100+ | 环境/作用域/键名别名哈希表，一次 token 扫描 |
| 时间变量 | 100+ | 时间别名哈希表 + 按需格式化 |
| 随机变量 | 100+ | 随机类别哈希表 + 按需生成 |

非 HTTP 请求变量目标解析延迟为 5ms 以内；引擎会在无变量字符或无变化时提前停止，避免 52 层嵌套空转。大规模变量使用哈希查找，不会对 520+ 变量逐个 `replace_all` 全量扫描。

## 注意事项

- 文本类截取和长度以 UTF-8 字节为单位，不按 Unicode 字符宽度计算。
- 参数变量内部若含 `]`，请使用固定常量变量辅助嵌套。
- 嵌套变量会多轮解析，当前最大 52 层；复杂组合建议控制长度，避免无限展开。
- 五大 100+ 类别与 520+ 固定变量族均使用 token 扫描 + 哈希/前缀分派，不会循环枚举几百个 `replace_all`。
- 5ms 是非 HTTP、非磁盘慢查询场景的目标值；实际速度会受回复长度、嵌套层数、CPU、存储后端影响。
